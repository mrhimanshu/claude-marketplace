Generate clear, comprehensive documentation for the selected code or project. Adapt the output based on context:

**For a function/class/module:**
- Purpose and behavior description
- Parameters with types and descriptions
- Return values
- Usage examples
- Edge cases and error handling notes

**For a project/directory:**
- Overview of what the project does
- Architecture and key components
- Setup and installation instructions
- Configuration options
- API reference (if applicable)
- Common usage patterns

**Documentation style:**
- Write for developers who are new to this codebase
- Use clear, concise language
- Include runnable code examples
- Document the "why" not just the "what"
- Use appropriate format (JSDoc, docstrings, markdown) based on the language

Output the documentation ready to use — either inline code comments or a markdown document.
